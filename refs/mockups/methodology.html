Phase 0: Document Classification
classify_decisions.py • Claude Sonnet 4.5
▼
Purpose

Before extracting citations, each document must be classified as a judicial decision or non-decision. The Climate Case Chart database includes various document types (pleadings, motions, administrative documents), but only judicial decisions are relevant for citation analysis.

Two-Stage Classification Strategy
1
Document Title Heuristic (Fast Path)
Check if the last word of the Document Title is "judgment" or "decision". If matched, classify as decision with 100% confidence. No API cost. This provides immediate classification for clearly-labeled documents.
2
LLM Classification (Fallback)
For documents where the title is inconclusive, send to Claude Sonnet 4.5 for intelligent classification. Uses structured XML prompt with explicit inclusion/exclusion criteria. Cost: ~$0.003 per document.

Phase 1: Source Jurisdiction Identification
extract_citations.py • Database Lookup
▼
Purpose

Identify the jurisdiction (country/region) of the citing court. This is essential for determining whether a citation is domestic or cross-jurisdictional. The source jurisdiction is extracted from the Case.geographies field in the database.

Algorithm
get_source_jurisdiction() 
def get_source_jurisdiction(geographies_string: str) -> str:
    """
    Extract primary jurisdiction from Geographies field.
    
    INPUT: "United States; California; Washington, D.C."
    ALGORITHM:
        1. Split by semicolon
        2. Take first value (country level)
        3. Handle international tribunals
    OUTPUT: Primary country string
    """
    if not geographies_string:
        return "Unknown"
    
    parts = [p.strip() for p in geographies_string.split(';')]
    primary = parts[0]  # Country level only
    
    # Check if international
    if primary in ['International', 'INTL', 'World']:
        return "International"
    
    # Normalize jurisdiction
    return normalize_jurisdiction(primary)

Phase 3: Origin Identification
3-Tier Approach • Dictionary + LLM + Search
▼
Purpose

Identify the country/jurisdiction of origin for each extracted citation. Uses a tiered approach to balance cost, speed, and accuracy.

3-Tier Architecture
T1
Tier 1: Dictionary Lookup (Fastest, Free)
Check cached results first. Then search KNOWN_FOREIGN_COURTS (80+ courts) and LANDMARK_CLIMATE_CASES (20+ cases) dictionaries. If found, return with 95% confidence. Zero API cost.
T2
Tier 2: Claude Sonnet Analysis
For citations not found in dictionaries, use Claude Sonnet 4.5 to analyze court name, citation format, case name patterns, and legal system indicators. Cost: ~$0.01 per citation.
T3
Tier 3: Web Search (Fallback)
For obscure or uncertain cases, web search can be used as a final fallback. Currently placeholder for future implementation.

Phase 4: Citation Type Classification
Rule-Based Classification
▼
Purpose

Classify each citation based on the relationship between source jurisdiction and cited jurisdiction. This uses deterministic rule-based logic, requiring no LLM calls.

Classification Logic
classify_citation_type() 
def classify_citation_type(source_jurisdiction, source_region,
                          case_origin, case_region):
    """
    Phase 4: Classify citation type based on source and origin.
    
    OUTPUT: (citation_type, is_cross_jurisdictional)
    """
    # Normalize jurisdictions
    source_norm = normalize_jurisdiction(source_jurisdiction)
    case_norm = normalize_jurisdiction(case_origin)
    
    # Handle unknowns
    if case_origin == 'Unknown' or case_region == 'Unknown':
        return 'Unknown', False
    
    # Same jurisdiction = Domestic (NOT cross-jurisdictional)
    if source_norm == case_norm:
        return 'Domestic', False
    
    # Different jurisdictions = Cross-jurisdictional
    is_cross_jurisdictional = True
    
    # Classification logic
    if source_region == 'International' or case_region == 'International':
        if source_region == 'International' and case_region == 'International':
            return 'International Citation', True
        elif source_region == 'International':
            return 'Foreign Citation', True
        else:
            return 'International Citation', True
    
    # Both are national courts (different countries)
    return 'Foreign Citation', True
Zero API Cost
Phase 4 uses purely deterministic rules based on the origin data from Phase 3. No LLM calls required, making classification instant and free.

Sixfold Citation Classification Schema
sixfold_classification_complete.sql
▼
Overview

The sixfold classification schema provides nuanced categorization of cross-jurisdictional citations by considering both the direction of citation (national vs. international) and the membership relationship between source and cited jurisdictions in international court systems.

1
Foreign Citation
National court citing another national court from a different country. The most common form of transnational judicial dialogue.
Direction: National → National
Example: UK court citing Dutch Urgenda case
2
International Citation
National court citing an international tribunal where the citing country is a member state with binding jurisdiction.
Direction: National → International (Member)
Example: UK court citing ECtHR case
3
Foreign International Citation
National court citing an international tribunal where the citing country is NOT a member state (persuasive only).
Direction: National → International (Non-Member)
Example: US court citing IACtHR case
4
Inter-System Citation
International tribunal citing another international tribunal. Represents dialogue between international legal systems.
Direction: International → International
Example: ECtHR citing ICJ advisory opinion
5
Member-State Citation
International tribunal citing a national court from one of its member states.
Direction: International → National (Member)
Example: ECtHR citing German Constitutional Court
6
Non-Member Citation
International tribunal citing a national court from a non-member state.
Direction: International → National (Non-Member)
Example: IACtHR citing Indian Supreme Court
International Court Membership

The sixfold classification requires mapping international courts to their member states. This is implemented in the international_court_jurisdiction support table.
